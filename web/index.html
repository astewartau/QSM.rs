QSM.rs

The Rust engine for Quantitative Susceptibility Mapping. A from-scratch library implementing 40+ published algorithms across the full reconstruction chain — the same engine that powers QSMxT and QSMbly.

API documentation → View on GitHub

Get started

Add the crate as a git dependency, then reconstruct in a few lines. There's a high-level pipeline API and a low-level building-block API — both shipped as runnable examples.

Cargo.toml
# A single git dependency — no system libraries
[dependencies]
qsm-core = { git = "https://github.com/astewartau/QSM.rs" }

# optional: multi-threading and SIMD
# features = ["parallel", "simd"]
high-level pipeline
let field = run_field_mapping(&phases, Some(&mags),
    &mask, &meta, &cfg, &mut cb)?;
let bg = run_bg_removal(&field.b0_field_ppm,
    &mask, &meta, &cfg, &mut cb)?;
let chi = run_dipole_inversion(&bg.local_field_ppm,
    &bg.eroded_mask, &meta, &cfg, mag, &mut cb)?;

Part of the QSMxT Ecosystem

QSM.rs is the shared core. If you're not writing Rust, you probably want one of these front doors — they all call straight into this library, so results match exactly.
